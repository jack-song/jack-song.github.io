RTFM

I thought this would be a story about finding a flaw in the great google machine - instead it’s about being prudent when working with dependencies.

I am currently working on an app for Kenyan users. With SMS functionality, we have validation for local phone numbers. While we were testing, I realized that my number is considered invalid by google’s libphonenumber library. It has a 254798 prefix presumably newly introduced as I had picked up my SIM at the airport days earlier.

I check the library and searched for “kenya”. After a quick scan, a comment stands out: “Prefix 25476 added for Airtel Kenya based on open-source bug report.” Followed by “other prefixes from Wikipedia.” I take a look and Wikipedia does not contain a prefix for my number. Everything made sense. There’s no way google has the resources to keep up with all the technical changes in the world - and certainly not the developing areas where documentation is poor and good practice is as well respected as traffic regulations.

Excitedly, I thought I would get to file a bug report against a google repository. Nervously, I realized we’d have to roll our own phone number validation to release on schedule. So I hammer out a horrific switch case method and throw it in. But as I go to file the report, I read the issue guidelines, which includes this step: “Is the issue reproducible using the demo?,” and this step: “Your issue may be resolved by upgrading to the latest version of the library.” Mmmmmm. I try my number in the online demo. It works. I go back to the code file. Appended at the very end of file: “25479|Safaricom.” Sure enough, searching the issues gives this.

I rollback and just update the library. Maybe I should submit a change to Wikipedia. So even if you’re using a library for a volatile domain - give them the benefit of the doubt and first RTFM. An argument can also be made that in a domain this volatile, it could be wise to use a less strict form of validation in the first place, depending on the business value of the feature always working vs. the consequences of allowing an illegal value.
